Удаленное подключение

Иногда возможно подключиться к системе из одного места, но невозможно из другого.

Для того чтобы удаленное подключение заработало без настроек проброса портов, DDNS, VPN и т. д., система пытается открыть туннель (UDP Hole Punch) на стороне клиента и на стороне сервера. Успех зависит от многих факторов, например: модель/прошивки маршрутизатора, тип сети ISP (сотовая, кабельная, wi-fi), сетевая инфраструктура, страна и т. д.
Несмотря на то, что обычно это работает хорошо, этот функционал может не работать в конкретном случае.

Проблема может быть решена ручным пробросом порта. Настройка DDNS никогда не требуется (даже при динамическом публичном IP-адресе).

Объединение облачных систем

Объединение облачных систем на данный момент не поддерживается. Локальные системы могут быть объединены с облачными, в любой момент времени. Поэтому для того, чтобы объединить две облачные системы, одну следует отключить от {{PRODUCT_NAME}}.

Вход в систему не в сети

Учетные данные {{PRODUCT_NAME}} нельзя использовать для подключения к системе, которая не подключена к сети Интернет. Чтобы подключиться к такой системе локально (например с именем пользователя «admin»), следует использовать локальные учетные данные пользователя.
