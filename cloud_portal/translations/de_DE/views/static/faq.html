Remoteverbindungen

Manchmal ist es möglich sich zum System von einem Netzwerk/Ort zu verbinden und von einem anderen nicht.

Um eine Remoteverbindung ohne Portweiterleitung, DDNS, VPN, etc. herzustellen, muss die Software das "UDP-Loch" sowohl auf der Clientseite als auch auf der Serverseite überwinden. Der Erfolg hängt von vielen Faktoren ab, wie z. B. dem/der Routermodell/Routerfirmware, dem ISP, dem Netzwerktyp (Kabel, Wifi), der Infrastruktur, etc.
Auch wenn es im allgemeinen gut funktioniert, kann es in Einzelfällen zu Fehlfunktionen kommen.

Durch manuell konfigurierte Portweiterleitungen kann das Problem möglicherweise behoben werden. DDNS wird nie benötigt (auch mit nicht-statischen-öffentlichen IP-Adressen).

Cloudsystem zusammenführen

Die Zusammenführung von Cloudsystemen wird derzeit nicht unterstützt. Das lokale System kann zu jeder Zeit mit der Cloud zusammengeführt werden. Um zwei Cloudsysteme zusammenzuführen, sollte eines von {{PRODUCT_NAME}} getrennt werden.

Offline-Login

Die Anmeldeinformationen von {{PRODUCT_NAME}} können nicht verwendet werden, um sich zu einem System zu verbinden, welches momentan keine Internetverbindung hat. Um eine Verbindung zu diesem System lokal herzustellen (wie z.B. "admin"), sollten lokale Benutzeranmeldeinformationen verwendet werden.
