Known limitations in {{PRODUCT_NAME}}

Remote connectivity

Sometimes it's possible to connect to System from one network/place and impossible from another.

In order to get remote connectivity to work without port forwarding, DDNS, VPN etc, software has to "punch the UDP hole" on the client side as well as on the server side. Success depends on many factors such as router model/firmware, ISP network type (cell, cable, wi-fi) and infrastructure, country etc.
In spite of working well in general, it may not work in a particular case.

The issue could be fixed by manual port forwarding setup. DDNS is never needed (even with non-static public IP address).

Cloud System Merge

Cloud Systems merging is not supported at the moment. Local System can be merged with the cloud one at any given time. So in order to merge two cloud Systems one should be disconnected from {{PRODUCT_NAME}}.

Offline Login

{{PRODUCT_NAME}} credentials cannot be used to connect to System which has no Internet at the moment. In order to connect to such System locally (such as "admin") local user credentials should be used.
