Reviewing and Publishing Revisions

Sending a review request triggers an email notification to the Portal Manager titled "Version xxx is ready to be reviewed," letting the Portal Manager know there are changes ready for review. Content Management > Revisions provides a table of the each numbered revision that has been published (old), the current live version (current) and the version most recently sent for review (in review), grouped by page name:

•This table can be sorted by clicking on one of the column headers (product, customization, date created, person who created the changes, date accepted, or person accepted by).
•There is a search tool that applies to all fields in the table.
•Revisions can be filtered by product or customization using the filter list to the right of the table.

To Publish a Version

The Portal Manager goes to the Revisions page to review or edit updates. Revisions can be published as-is, edited and then published, or edited and then sent to others for review.

1.Click the Review Version button in the email. The Cloud Portal will open to the revision number indicated in the email. (Or, click on Revisions from the dashboard and click on the Review button for the in review version.)
2.Each field change is listed separately. If desired, use the edit link under each change to change the revision.
3.To preview the revision, click Generate Preview and then Open Preview. The changes will open in a new tab with a small "preview" indicator in the upper-right corner of the page.
4.To approve the revision, click Publish to push the revision out to the live site.

Preview and publish operations may take some time to complete.

5.If there are changes to the branding color scheme (skin), you must click Force Update to deploy them.
6.Once updates are published, click the View Site link and then the About {{Product Name}} footer at the bottom of the landing page as a shortcut for viewing pages.

Forces Updates

A change to the skin requires a "forced update" because all static files on the hard drive must be updated. Use with caution - depending on the size of your site this process may take a long time.

IMPORTANT!  Do not close the page until force update is complete and the confirmation displays.

If Changes Do Dot Display

There are a few steps you can take if updates are not displayed when you generate a preview. Make sure to refresh the page, and try viewing the page in a browser window that does not store cache (ex. an incognito window). Try clicking "Force updates". Alternately, you can inspect the source code and disable cache.
