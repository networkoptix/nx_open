Cloud Portal Customization Management System (CMS) Overview

The customization management system (CMS) is a cloud service that allows authorized partners to customize a Cloud Portal with their own branding and content. Certain changes, however, can only be performed by Network Optix directly or by a partner who has been granted special advanced editing permission.

Elements that can be customized by a partner from the Cloud Portal CMS include:

•Logos and images
•Default color (for header, footer, background, text, etc.), called the "skin"
•Product and company names
•Display language
•Landing page content
•Links for support, emails, company website, etc.
•Users with the appropriate access rights can also view, sort, filter, and export the list of registered Cloud users

Site Administration

The first step to using the CMS is to contact Network Optix to arrange for access rights for the staff members who will have authority to view, use, or make changes in the customization management system.

There are three levels of authority – Reporters, Content Managers, and Portal Managers.

Reporters

Users in this group can view history for releases and patches, usually to report on problems or acquire the download link and password for a particular build number.

To access the page with all release, patch and beta Release Notes, go to {{portal url}}/downloads/history.

Content Managers

Users in this group can edit but not publish content.

Portal Managers

Users in this group can edit and publish content, and view and download customer account records.

Update Process

There are three steps to a site update:

1. The Content Manager enters and previews changes.

2. Once editing is complete, content is saved and sent to the Portal Manager for review. The Content Manager gets a confirmation that changes have been saved and a new version has been created under a specific Revision number.

3. The Portal Manager reviews changes and publishes them.

Portal edits can be previewed in draft format at any time without disturbing live pages.

Revision Numbering

Sending a set of changes for review generates a numbered Revision (or version) which logs the exact changes made, by whom, and when. Revision tracking makes it possible to view any published version. Note that the content of revision requests is additive. The Portal Manager receives an email notification for each numbered review request, but only the latest request is shown on their revisions page, and that version will contain all changes that have been sent since the last time changes were published.  For example, the current live site is revision 117. Versions 118, 119 & 120 are sent for review but not are published. Revision 121, however, is published – and will contain all the changes made in versions 118 - 120.

Remember, all revisions since the last published set are bundled into the most recent version.
