Editing Pages

Content Management > Pages provides a table of the pages that can be customized.

This table can be sorted by clicking on one of the column headers, and there is a search tool that applies to all fields in the table.

•This table can be sorted by clicking on one of the column headers (product, name, description).
•There is a search tool that applies to all fields in the table.
•Pages can be filtered by product using the filter list to the right of the table.

To Edit Page Content

Click on the Edit content button for a specific page to enter changes.

•Required fields are shown in red. If you click in an editable text field, the outline will change to blue.
•The database label (e.g. %SUPPORT_LINK%), field content (e.g. "App ID for iOS mobile application"), and any limits (ex. required file type, maximum number of characters) are shown for each field.
•You can use Cloud Portal database labels as variables in any field.
•click on Link to Page to view the current version of a page.

Saving Changes

Once changes are entered you have several options:

•Save and Preview - saves the latest changes and generates a preview page. You will see the message "Changes have been saved. Preview has been created." Click the Open Preview button at the bottom of the page to view the updated page in a new browser tab.
•Save Draft - saves a draft of pages changes before publishing so you can return to work in progress. You will see the message "Changes have been saved."
•Send for Review - saves a draft of the latest content and sends it for review. You will see the message "Changes have been saved. A new version has been created." Note that this may take some time if the branding page has been updated. Once changes are saved a confirmation screen displays showing the revision number and details of the changes.

•Reset - undoes all changes.

HTML Editor Fields

Certain page fields are also HTML editors and therefore allow for text formatting and the use of hyperlinks. Click on the Edit HTML source icon to open an editor that shows HTML code. You can also paste formatted Word content into these fields intact using the Paste from Word icon.

Selecting a Language

Some pages have a language selection option. This setting controls the language used in certain elements of the Cloud Portal interface, such as the log in dialog, revision notice emails, or the shortcut tabs at the bottom of the landing page. Note that this setting does not translate the contents of all fields on a page. Selecting a language for a given page only changes a predefined set of interface elements. For the entire page to be in a given language, the content of user-editable fields must be entered in that language.

It is important to note that a separate template is saved for each language option that you select, and each template must be saved, edited, and reviewed independently.

Additional language support for the Cloud Portal is part of your customization agreement with Network Optix. Though you may see many translation options when editing page content, only specific language options will be available according to your customization contract.

To View Page Change History

Click on a page name to open its Change page, which displays the database values for that page. Click on the History button to see a log of field changes. This page can only be edited by Network Optix.

Fields shown are:

•Product – indicates the active site.
•Description  – shows the description used in the Pages list.
• URL – URL for the page.
•Translatable  – if a page is translatable (green check icon ) you will be able to select from a menu of available languages on its edit page. If it is not translatable it will have a red x icon .
•Name – the name displayed in the Pages list.
