Generischer Ereignisgenerator

Ein generisches Ereignis ist eine Methode zum Senden von Ereignissen von Drittsystemen an ein System, mit der Systemaktionen in der Ereignisse & Regeln Engine ausgelöst werden können. Der generische Ereignisgenerator ist ein Tool, mit welchem generische HTTP-Ereignisaufrufe erstellt werden können.

Server-API

Die Server-API stellt ein Standardset von API-Aurufen für 3rd-Party-Systeme bereit. Es gibt Entwicklern die Möglichkeit auf fast jede Systemfunktion im normalen Client zugreifen: Abfrage/Verwaltung von Systemressourcen (Server, Kameras, Benutzer, Berechtigungen), Live- oder aufgezeichnete Videos aus dem System ziehen, Ereignisse generieren, Regeln erstellen, PTZ-Kameras steuern und vieles mehr.

Kamera Detail Panel

Zeigt zusätzliche entwicklerfreundliche Informationen für Kameras in der Webadmin-Oberfläche an:

  • Kamera-ID
  • Links für alle verfügbaren Videostreams
  • Video direkt vom Browser exportieren

Um das Kamera Detail Panel zu öffnen - navigieren Sie zur Webadmin-Oberfläche, wählen Sie Ansicht, wählen Sie die Zielkamera aus, klicken Sie auf das Einstellungssymbol (Zahnrad) und wählen Sie dann "Kameradetails"

Video Source Integration SDK

Die Video Source Integration SDK bietet die Möglichkeit praktisch jede Live- oder aufgezeichnete Videoquelle (IP-Kameras, NVRs, DVRs, etc.) ins System zu integrieren. SDK ermöglicht es Methoden zur Entdeckung, Anzeige, Analyse und Videoufzeichnung zu erstellen. Es erlaubt auch die Integration von Geräte-I/O-Ports und Bewegungserkennungsinformationen.

Storage SDK

Storage SDK bietet die Möglichkeit potentielle Speicher ins System zu integrieren. Es ermöglicht Entwicklern das Lesen oder Schreiben auf jeden Speicherort: Lokal, Remote und sogar über die Cloud. Das Erstellen eines Speicherplugins erfordert die Implementierung von Standardfunktionen wie: I/O-Stream, falls Datei vorhanden ist, Datei löschen, Liste der Dateien im Ordner usw. Storage SDK enthält auch ein Beispiel für die Verwendung eines FTP-Servers als Speicherort.
