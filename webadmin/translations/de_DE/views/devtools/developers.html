Generischer Ereignisgenerator

Ein generisches Ereignis ist eine Methode zum Senden von Ereignissen von Drittsystemen an ein System, mit der Systemaktionen in der Ereignisse & Regeln Engine ausgelöst werden können. Der generische Ereignisgenerator ist ein Tool, das Entwicklern und Integratoren hilft, HTTP-Ereignisaufrufe zu erstellen und zu testen.

Server API

Eine REST-API, die Entwicklern die Möglichkeit gibt, auf fast alle im regulären Client verfügbaren Systemfunktionen zuzugreifen: Abfragen/Verwalten von Systemressourcen (Server, Kameras, Benutzer, Berechtigungen), Herausziehen von Live- oder aufgezeichneten Videos aus dem System, Erzeugen von Ereignissen, Erstellen von Regeln, Steuern von PTZ-Kameras und mehr.

Kamera Detail Panel

Zeigt zusätzliche entwicklerfreundliche Informationen für einzelne Kameras auf der Registerkarte Ansicht der Webadmin-Oberfläche an:

  • Camera ID
  • Links für alle verfügbaren Videostreams
  • Video direkt vom Browser exportieren

Um das Kamera Detail Panel zu öffnen - navigieren Sie zur Webadmin-Oberfläche, wählen Sie Ansicht, wählen Sie die Zielkamera aus, klicken Sie auf das Einstellungssymbol (Zahnrad) und wählen Sie dann "Kameradetails"

Video Source SDK

Das Video Source SDK bietet die Möglichkeit, nahezu jede lebende oder aufgezeichnete Videoquelle (IP-Kameras, NVRs, DVRs, etc.) in das System zu integrieren und ermöglicht es, Methoden zum Erkennen, Anzeigen, Analysieren und Aufzeichnen von Videos zu erstellen. Es ermöglicht auch die Integration von I/O-Ports und Geräten sowie das Abrufen von Bewegungserkennungsinformationen von Drittanbietergeräten.

Storage SDK

Das Storage SDK ermöglicht es Entwicklern, von jedem beliebigen Speicherort zu lesen oder zu schreiben: lokal, remote und sogar in der Cloud. Die Erstellung eines Speicherplugins erfordert die Implementierung von Standardfunktionen wie: I/O-Stream, wenn Datei vorhanden, Datei löschen, Liste der Dateien im Ordner, etc. Das Storage SDK enthält ein Beispiel für die Verwendung eines FTP-Servers als Speicherort.
