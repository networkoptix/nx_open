Generador de eventos genéricos

Un evento genérico es un método para enviar eventos desde sistemas de terceros a un sistema que se puede usar para desencadenar acciones del sistema en el motor de eventos y reglas. El Generador de eventos genérico es una herramienta que ayuda a los desarrolladores e integradores a construir y probar llamadas de eventos genéricos de HTTP.

Servidor API

Una API REST que brinda a los desarrolladores la capacidad de acceder a casi todas las funciones del sistema disponibles en el cliente habitual: consultar / administrar recursos del sistema (servidores, cámaras, usuarios, permisos), extraer video en vivo o grabado fuera del sistema, generar eventos, crear Reglas, control de cámaras PTZ, y más.

Herramienta de prueba de API (nueva) : documentación interactiva y herramientas para probar las llamadas a la API
API Registro de cambios : una lista de los últimos cambios en la API desde la última versión
API Documentation - una lista completa de las llamadas a la API del servidor disponibles

Panel de detalles de la cámara

Muestra información adicional fácil de usar para el desarrollador para cámaras individuales en la pestaña de Vista de la interfaz de administración de red:

  • ID de cámara
  • Enlaces para todos flujos de vídeo disponibles
  • Exportar vídeo directamente desde el navegador

Para abrir el Panel de Detalles de la Cámara: navegue hasta la interfaz Web Admin, elija Ver , seleccione la cámara de destino, haga clic en el ícono de configuración (gear), luego elija "Detalle de Cámaras"

Fuente de video SDK

The Video Source SDK provides the ability to integrate almost any live or recorded video source (IP Cameras, NVRs, DVRs, etc) into System and makes it possible to create methods for discovering, displaying, analyzing and recording video. It also allows integrating I/O ports and devices, as well as pulling motion detection information from 3rd party devices.

SDK de Almacenamiento

The Storage SDK allows developers to read from or write to any storage location: local, remote, and even cloud one. Creating a storage plugin requires implementing standard functions such as: I/O stream, if file exist, delete file, list of files in the folder, etc. The Storage SDK contains an example for using an FTP server as a storage location.
