一般的なイベント ジェネレーター

一般的イベントとは、サードパーティ製のシステムからイベントをシステムへ送信する方法です。イベント＆ルール でシステムアクションを起動するために使用できます。 一般イベントジェネレーターは、開発者やインテグレーターがHTTP一般イベント呼び出しを構築し、テストするのに役立つツールです

Server API

正規クライアントで利用可能なほとんどすべてのシステムの機能にアクセスすることができます：クエリ/管理システムリソース (サーバー、カメラ、ユーザー、承認)、システムからライブまたは録画した映像を引き出す、イベントを発生させる、ルールを作成する、PTZ カメラを制御するなど。

APIテストツール（新規） - API呼び出しをテストするための対話型資料およびツール
API変更ログ - 前回のリリース以降のAPIの重大な変更点のリスト
APIドキュメント - 利用可能なサーバーAPI呼び出しの完全なリスト

カメラ詳細パネル

Web管理インターフェースの [表示] タブに、個々のカメラに関する追加の開発者向け情報を表示します。

  • Camera ID
  • 利用可能なすべてのビデオストリームのリンク
  • ブラウザからビデオのエクスポート

カメラの詳細パネルを開くには - Web Adminインターフェイスに移動し、表示を選択 、ターゲットカメラを選択し、設定アイコン（歯車）をクリックしてから、「カメラの詳細」を選択します

ビデオソースSDK

ビデオソースSDKは、ほぼすべてのライブまたは録画された映像ソース (IPカメラ、NVR、デジタルビデオレコーダーなど) をシステムに統合する機能を提供し、検出、表示、分析、ビデオ録画するための方法を作成できるようにします。 また、I/Oポートとデバイスを統合するだけでなく、サードパーティのデバイスから動体検知の情報を引き出すこともできます。

SDK を取得

ストレージ SDK

ストレージSDKは、開発者が任意の保存場所から読み取ることあるいは書き込むことを許可します：ローカル、リモート、クラウド ストレージプラグインを作成するには、以下の標準機能を実装する必要があります：I/Oストリーム、ファイルが存在する場合にはファイルの削除、フォルダ内のファイルのリストなど。 ストレージSDKには、保存場所としてFTPサーバーを使用する例が含まれています。
