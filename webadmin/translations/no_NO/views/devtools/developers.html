Generisk hendelse generator

En gnerisk hendelse er en metode for å sende hendelser fra tredjepartssystemer til et system, som kan brukes til å utløse systemhandlinger i hendelser og regler-motoren. Generic Event Generator er et verktøy som hjelper utviklere og integratorer til å bygge og teste HTTP Generic Event calls.

Server API

Et REST API som gir utviklere muligheten til å få tilgang til nesten alle systemfunksjoner som er tilgjengelige i den vanlige klienten: spørre/administrere systemressurser (servere, kameraer, brukere, tillatelser), hente live eller innspilt video ut av System, generer hendelser, sett opp regler, kontroller PTZ kameraer og mer.

Kameradetaljer Panel

Vise utvikler-vennlig informasjon for individuelle kameraer i Web-admingrensesnittets Vis-faneblad:

  • Kamera-ID
  • Lenker for alle tilgjengelige videostrømmer
  • Direkte videoeksport fra nettleser

For å åpne Kameradetaljer-panelet - naviger til Web Admin-grensesnittet, velg Vis , velg målkamera, klikk på innstillingsikonet (gir), og velg deretter "Kameradetaljer"

Videokilde SDK

Videokilden SDK gir muligheten til å integrere nesten hvilken som helst levende eller innspilt videokilde (IP-kameraer, NVRer, DVRer, osv.) i Systemet og gjør det mulig å lage metoder for å oppdage, vise, analysere og spille inn video. Det tillater også å integrere I/O-porter og enheter, samt å hente bevegelsesdeteksjon informasjon fra tredjepartsenheter.

Lagrings SDK

Lagrings-SDK lar utviklere lese fra eller skrive til alle lagringssteder: lokal, ekstern og til og med sky en. Å utvikle en lagringspluggin krever implementering av standardfunksjoner som: I/O-strøm, hvis filen eksisterer, slett filen, listen over filer i mappen osv. Lagrings-SDK inneholder et eksempel på bruk av en FTP server som lagringsplass.
