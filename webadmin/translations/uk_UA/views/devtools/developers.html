Генератор довільних подій

Довільний подія - спосіб відправки подій із сторонніх систем. Довільні події можуть використовуватися для запуску системних дій в механізмі подій і правил. Генератор довільних подій - інструмент, який виконує виклики довільних подій HTTP.

Генератор довільних подій

API сервера

API сервера забезпечує стандартний набір API викликів для зовнішніх систем. Це дає можливість розробникам отримувати доступ до майже кожної функції системи, доступної в звичайному клієнті: запиту і управління системними ресурсами (сервери, камери, користувачі, повноваження), отримання відео в реальному часі або архіву з системи, генеріраціі подій, створення правил, управління PTZ -Камера і багато чому іншому.

Інструмент тестування API (новий)              - інтерактивна документація з інструментами для тестування викликів API
             API-журнал змін               - список змін в API, що можуть торкнутися поточний функціонал
             Документація API
             Документація з налагодження сервера

Панель відомостей про камеру

Показує додаткову інформацію по камерах, корисну для розробників, в інтерфейсі Web Admin:

  • Ідентифікатор камери
  • Посилання для всіх доступних відеопотоків
  • Прямий експорт відео з браузера

Щоб відкрити панель відомостей про камери - перейдіть в інтерфейс веб-адміністратора, виберіть Вид, виберіть бажану камеру, клацніть значок налаштувань (шестерня), потім виберіть "Відомості про камеру"

SDK для інтеграції джерел відео

SDK надає можливість інтеграції практично будь-якого джерела живого або записаного відео (IP-камер, NVR, цифрових відеореєстраторів, і т. д.) в систему. SDK дозволяє створювати методи для виявлення, відображення, аналізу та запису відео. Також можливо інтегрувати порти пристроїв введення-виведення і дані виявлення руху.

SDK для інтеграції сховищ

SDK зберігання даних надає можливість інтеграції потенційних сховищ в систему. Це дозволяє розробникам виконувати операції читання/запису в будь-яке місце зберігання: місцеве, віддалене і навіть хмарне. Створення плагіна зберігання даних вимагає реалізації стандартних функцій таких як: потоковий ввід/вивід, перевірка існування файлу, видалення файлу, отримання списку файлів в папці і ін. SDK також містить приклад використання FTP-сервера в якості сховища.
