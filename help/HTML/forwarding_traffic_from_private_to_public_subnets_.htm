Forwarding Traffic from Private to Public Subnets via Mediaproxy Component

Since version 1.1 DW Spectrum provides media traffic proxy feature which enables easier solution integration into enterprise.

Proxy traffic is crucial if needed to access a media server that is located in different private network:

In this network diagram the client cannot access Media Server as it is located to the different network. This is solved by separate component (Media Proxy) that is installed and run along with Enterprise Controller. Media Proxy is listening on the specific Proxy Port (that is specified during the installation).

If the client is unable to connect to Media Server it acquires Proxy Port from EC then trying to connect to Media Server via EC IP address followed by Proxy Port.

If Media Proxy receives a request on this port it passes these requests to Media Server. All data from Media Server are passed to Client on the same port.

Thus in order to allow access to the private networking it is necessary to forward two ports:

•EC Port.

•Proxy Port.

No additional configuration is needed.
