Requirements for Addressing IP

In order for Media Server to discover and operate cameras properly, the cameras must be located on the same subnet. Use one of the following options:

•Automatic (DHCP)

1.Set up a DHCP server in private (cameras) network.

2.Configure cameras to obtain IP automatically (via DHCP).

Note: even though IP changes are handled properly, the recording will not be performed when a camera IP is changed (might take up to a minute).

3.Media Server’s IP should be permanent. It is necessary to configure DHCP for manual IP bound to server’s MAC address.

4.Client IP addresses can also be assigned by DHCP. DHCP server should be installed in public network as well.

•Manual. Cameras’ IP addresses are configured manually.

To configure camera IP settings, use one of the following:

•Use a camera manufacture’s proprietary software to discover cameras and change their IP addresses or switch them to DHCP.

•Configure a camera directly from its web page:

5.Use ONVIF discovery tool (http://sourceforge.net/projects/onvifdm/) to obtain a camera IP.

6.Change your network settings to the same subnet (i.e. if a camera’s IP is 169.175.169.100, a user must setup static address: IP 169.175.169.100 and mask 255.255.255.0).

7.Open the camera web page (http://169.175.169.100).

Please contact the camera manufacturer if you are still experiencing difficulties.

Note: Arecont Vision camera IP addresses are automatically discovered by DW Spectrum Media Server. IP conflicts can also be resolved automatically.
