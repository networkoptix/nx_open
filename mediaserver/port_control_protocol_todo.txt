
- mediaserver uses port contorl protocol to establish port forwarding

- server has to verifies that it is available from outside
    - by comparing its public address received from mediator to address provided by router

- mediaserver reports its public endpoint address to mediator
    - mediator should try to connect to mediaserver to verify public endpoint availability before reporting it to others

- when someone asks mediator for connection to mediaserver, mediator provides mediaserver's public endpoint address
    - if caller failed to connect to server, mediator 
        - initiates hole punching for the caller
        - checks that mediaserver's public endpoint is still available
